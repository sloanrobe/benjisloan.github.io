Common CSS Properties

Fonts Borders Position

Fonts

Tag Options Description
<font-family> serif, sans-serif, Georgia, Helvetica Specifies the font for an element
<font-weight> normal, lighter, bold, bolder Specifies how thick or thin some font characters are displayed
<font-style> normal, italics, oblique Sets the style of the font
<font-size> 12px, 0.8em, smaller, 80% Sets the size of the font in different units or lengths
<text-transform> uppercase, lowercase, capitalize Controls the capitalization of the text

Borders

Tag Options Description
<border-color> blue, green, black, marroon Sets the color for the element's border
<border-style> solid, dotted, dashed, double Sets the design of the element's border
<border-width> normal, italics, oblique Sets the width of the element's border

Position

Type Description
static Default position. Set according to normal flow of page. Not affected by top, bottom, left, or right properties.
relative Element is positioned relative to its normal (i.e. static) position. Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.
absolute Element is positioned relative to the nearest positioned ancestor. If an absolute positioned element has no positioned ancestors, it uses the document body, and moves along with page scrolling. Absolute positioned elements are removed from the normal flow, and can overlap elements.
fixed Element is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position the element.
sticky Element is positioned based on the user's scroll position. A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport - then it "sticks" in place (like position:fixed).
